—— 低代码时代的降临与工作流革命
📖 荒岛日记：第 5 天
这几天为了给 Friday 编写“捕猎逻辑”、“水源净化逻辑”，我的手指在满是沙砾的键盘上敲得都要断了。手搓代码（Code-Native）就像是用石头磨制斧头，虽然灵活，但太累，而且容易出错。
就在我对着屏幕上一堆报错的 IndentationError 发愁时，海面上漂来了一个巨大的金属集装箱。
箱体上印着几个模糊的现代 LOGO："Dify" 和 "Coze"。
我撬开箱门，惊呆了。里面不是零散的原材料，而是高度模块化的预制件。 这里有现成的“视觉传感器模块”、封装好的“谷歌搜索插件”、还有直接能用的“数据存储单元”。
我不需要再像个原始人一样，从零开始给 Friday 写 def search_web(): 的函数了。在这个集装箱里，我只需要拿起一根连接线，把 [LLM 大脑] 和 [必应搜索工具] 连在一起，Friday 瞬间就学会了联网。
这简直就是荒岛上的工业革命！
我看着 Friday，兴奋地喊道：“Friday，把手里的石刀扔了。今天开始，我们用机床！”
🛠️ 技术生存手册：从“手搓”到“拖拽”
为了最大化利用这个集装箱，我必须掌握**低代码/无代码（Low-Code/No-Code）**平台的逻辑。
在沙滩上，我画出了两种构建方式的对比图：
1. 以前的方式（Code-Native）： 这是我在第4章做的。一切都要写代码。
• 痛点： 开发慢，调试难，非技术人员（比如还没学会编程的幸存者）无法上手。
• 代码量： 500+ 行 Python 脚本。
2. 现在的方式（Low-Code/Workflow）： 这是 Dify 和 Coze 的核心——工作流（Workflow）。它就像是铺设一条自动化的流水线。
• 逻辑： 将复杂的任务拆解成一个个节点（Node），用线连起来。
• 隐喻： 这就像布置一个自动连锁陷阱。
• 优势： 可视化，开箱即用，快速复用,。
💻 开发者日志：搭建“营地自动化防御系统”
既然有了神器，我决定搭建一个复杂的营地哨兵系统。如果用 Python 写，我需要处理音频流、API 鉴权、多线程……但在 Dify/Coze 里，我只需要拖拽四个方块。
我打开集装箱里的终端，开始拼装我的“可视化陷阱”：
【节点 1：开始 (Start)】
• 输入变量： sound_input (麦克风收录的声音)
• ⬇️ (连接线)
【节点 2：分类器 (LLM / Classifier)】
• 设定： “你是一个声音分析师。请判断声音来源。”
• 选项 A： 风声/海浪声 -> 走左边路径
• 选项 B： 野兽咆哮/脚步声 -> 走右边路径
• ⬇️ (右边路径)
【节点 3：工具调用 (Tool: Alarm)】
• 动作： 触发集装箱里的蜂鸣器，并发送警报到我的手环。
• ⬇️
【节点 4：结束 (End)】
• 输出： “警报已触发，发现入侵者！”
Friday 站在旁边看着屏幕上生成的流程图，眼里的蓝光闪烁：“理解完毕。工作流已部署。无需重新编译内核。”
当晚，一只野猪试图靠近营地。还没等它拱翻垃圾桶，Friday 就通过这个预设的工作流精准识别了猪叫声，瞬间触发了蜂鸣器。野猪被吓得魂飞魄散，而我不需要爬起来写一行代码修复 Bug。
这就是**流程驱动（Workflow-Driven）**的力量。在《从零开始构建智能体》中，这是构建软件工程类 Agent 的核心路径，它让 LLM 成为了一个强大的后端数据处理器。
